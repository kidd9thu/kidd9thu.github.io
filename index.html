Django数据库性能优化

发表于 2017-02-28   |   分类于 Python ， Django

Django的数据库层提供了很多方法来优化数据库访问的性能，这篇文章将集中介绍这方面的内容。

1.使用标准的数据库优化技术

在进行Django数据库访问性能优化之前，首先应该使用标准的数据库技术对其进行优化，比如给字段加索引，通过使用django.db.models.Field.db_index 来给一个Django模型类的字段加索引，设置这个属性字段的Field.db_index=True。

注：django对model中的fk和unique = True的字段将自动创建索引。

2.理解Django中QuerySet的工作机制对数据库访问优化至关重要：

1) QuerySet是懒加载的；
2) QuerySet只有在需要的时候才会被执行；
3) QuerySet执行后，其结果是保存在内存中的。

3.理解Django中QuerySet的缓存机制

QuerySet对调用方法是不执行缓存的。比如下面的两端代码，其中一个会被缓存，另一个不会：

>>> entry = Entry.objects.get(id=1)
>>> entry.blog # Blog对象会被从数据库查询出来
>>> entry.blog # 第二次访问的缓存对象，不会再次执行查询

但是对于调用的查询方法，是不会被缓存的：

>>> entry = Entry.objects.get(id=1)
>>> entry.authors.all() # 第一次会执行查询
>>> entry.authors.all() # 第二次会再执行一次查询

4.使用模板语言中的with标签

在视图模板中，针对QuerySet对象使用with标签

5.使用iterator()方法

对于缓存的QuerySet使用iterator()方法

6.将查询计算操作放在数据库中完成，不要在Python代码中完成。

1)使用filter,exclude完成查询过滤；
2) F()查询表达式;
3) 使用聚合函数来完成数据库聚合操作。

7.使用QuerySet.extra()明确的指出要查询的字段

8.使用原生SQL查询：对于复杂的数据库查询操作，使用原生SQL实现。

9.尽量一次查询出所有需要的信息

10.理解 QuerySet.select_related()方法，并尽量使用它。

11.只查询需要的数据

1）某些情况下，只使用 QuerySet.values()和 values_list()方法，查询出符合条件的结果集而不是完整的对象结果集；
2）某些情况下，只使用 QuerySet.defer() 和 only()过滤数据。

12.如果只是查询集合的数量，使用QuerySet.count()函数，而不是len(QuerySet)

13.如果想知道某个记录是否包含在某个结果集中，使用 QuerySet.exists()函数

14.避免过多的使用count() 和 exists()函数，比如下面的代码:

{% if display_inbox %}
	{% with emails=user.emails.all %}
	{% if emails %}
	
You have {{ emails|length }} email(s)

	{% for email in emails %}
	
{{ email.body }}

	{% endfor %}
	{% else %}
	
No messages today.

	{% endif %}
{% endwith %}
{% endif %}

这样做的好处是：
1) 如果display_inbox=False，则不必做查询；
2) 针对user.emails.all使用with标签，可以让数据被缓存起来使用；
3）if emails 会导致 QuerySet.__nonzero__()被调用，进而导致user.emails.all() 被调用；
4） emails|length会导致 QuerySet.__len__()被调用，这样可以使用缓存的QuerySet；
5) for循环可以通过迭代来使用缓存的结果。

15.对于批量更新和删除操作使用 QuerySet.update() 和 QuerySet.delete()

16.如果需要查询对象的外键，则使用外键字段而不是使用关联的对象的主键，比如：

>>>entry.blog_id #应该使用这种方式
>>>entry.blog.id #不要使用这种方式

[Algorithm]铺瓷砖问题

发表于 2016-05-13   |   分类于 Algotithm ， DP

题目地址：http://hihocoder.com/problemset/problem/1048

题目描述：

在一个N*M的格子里，我们现在有两种类型的砖块，1 2和 2 1，问一共有多少种方
案，可以将整个N*M的空间都填满。

该问题属于状态压缩DP问题。所谓状态压缩DP问题，简单来说就是使用比特位来表示问题的状态。

假设我们按行的从左到右，列的从上到下的顺序来依次铺设瓷砖。目前需要铺设位置为(i，j)的瓷砖，而且假设之前的位置都已经铺设好了。
则在该位置铺设瓷砖有三种情况：

  1. 不用铺砖了，在(i-1, j)的时刻已经被竖着铺上了，然后考虑(i， j+1)；
  2. 横铺砖，将(i, j+1)也铺上了，然后考虑(i, j+2)；
  3. 竖着铺砖，将(i，j)和(i+1，j)铺上一个竖立的瓷砖，然后考虑(i，j+1)。

我们用0，1来表示位置(i，j)所铺设的瓷砖的状态。如果在(i，j)我们横着铺设瓷砖，则将(i，j)和(i，j+1)位置的状态都置为1。如果我们
在(i，j)竖着铺设瓷砖，则将(i，j)位置的状态置为0，(i+1，j)位置的状态置为1。

根据这种定义，我们的问题就变成了求解d[i][j]。其中i代表行的标号，j代表该行的瓷砖铺设的状态。因为有M列，我们用Mbit来表示行的状态，
所以一行可能的状态为stateNum = 1<<M。对于i行的状态j，该状态是否合法至于i-1行的状态有关。求解d[i][j]，其实就是求解d[i-1][m]的和，其中m和j应该是两种相兼容的状态。兼容的意思是说，位置的状态符合我们定义的01的位置状态表示。于是我们可以得到d[i][j]的公式：

d[i][j] = sum(d[i-1][m]), m = [compatible(i,j), i = 0, ... , stateNum-1] i >= 1;
d[i][j] = 1 if islegal(j) else 0， i = 0;

判断第0行的状态是否合法的方法为：

假设我们当前铺设的是d[0, j]的第i个比特位，即第0行，i列：

  1. 如果x是1，那么 x + 1 也一定是1，然后测试到 x + 2；
  2. 如果x是0，那么直接测试下一个 x + 1；

判断当前行和上一行是否兼容的办法为：

假设我们当前铺设的是d[i,j]的第x个比特位，即第i行，x列：

  1. 如果(i,x)位置的值是0，那么(i-1,x)位上一定是1。因为不可能在同一列相邻的位置铺两个竖着的瓷砖的第一个1*1砖块，如果满足，
    则下一步考虑(i, x+1)，否则直接返回不兼容。

  2. 如果[i,x]位置的值是1。那么有可能有两种情况：

    1. (i-1, x) 是0，那么瓷砖一定是竖着铺设，下一步考虑(i, x + 1)；
    2. (i-1, x) 是1，如果是这样的话，那么(i, x)位置的瓷砖一定是要选择横着铺设，那么(i,x+1)也一定是1，并且(i-1,x + 1)一定是1(如果是0，就是竖着铺了)，如果不满足就返回不兼容，满足条件 就测试(i,x + 2)；

问题代码如下：

import java.util.Scanner;
public class Main {

    public static boolean testFirstLine(int i, int m){
        int j = 0;
        while(j < m){
            if((i & (1 << j)) == 0){
                j ++;
            }else{
                if((j == m - 1) || ((i & (1 << (1 + j))) == 0)){
                    return false;
                }else{
                    j += 2;
                }
            }
        }
        return true;
    }

    public static boolean testCompatible(int stateA, int stateB, int m){
        int i = 0;
        while(i < m){
            if((stateA & (1 << i)) == 0){
                if((stateB & (1 << i)) == 0){
                    return false;
                }
                i ++;
            }else{
                if((stateB & (1 << i)) == 0){
                    i ++;
                }else if(i == m - 1 || !(((stateA & (1 << (1 + i))) != 0) 
                        && ((stateB & (1 << (1 + i))) != 0) )){
                    return false;
                }else{
                    i += 2;
                }
            }
        }
        return true;
    }

    public static void main(String [] args){
        int n = 0, m = 0;
        Scanner in = new Scanner(System.in);
        n = in.nextInt();
        m = in.nextInt();
        int M = 1 << m;
        int [] [] f = new int[n][M];
        for(int i = 0; i < n; i ++){
            for(int j = 0; j < M; j ++){
                f[i][j] = 0;
            }
        }
        for(int i = 0; i < M; i ++){
            if(testFirstLine(i, m))
                f[0][i] = 1;
        }

        for(int i = 1; i < n; i ++){
            for(int j = 0; j < M; j ++){
                for(int k = 0; k < M; k ++){
                    if (testCompatible(j,k,m)){
                        f[i][j] += f[i-1][k];
                        f[i][j] %= 1000000007;
                    }
                }
            }
        }
        /*
        for(int i = 0; i < n; i ++){
            for(int j = 0; j < M; j ++){
                System.out.print(f[i][j] + " ");
            }
            System.out.println();
        }
        */

        System.out.println(f[n-1][M-1]);
    }
}

MySQL介绍(2)： 数据查询（Data Query Language，DQL）

基本查询操作

查询全部记录

SELECT * FROM table_name or view_name[, table_name or view_name]

查询指定列

SELECT column_name[, column_name, ...] FROM table_name or view_name[, table_name or view_name]

查询表中不重复的记录

SELECT DISTINCT column_name[, column_name, ...] FROM table_name or view_name[, table_name or view_name]

使用列别名查询

SELECT column_name AS alias_name [,column_name AS alias_name ] FROM table_name or view_name[, table_name or view_name]

对查询的记录进行算术运算

使用算术运算符对指定的列进行算术运算，算术运算符包括加减乘除。

使用连接符（||）连接字段

SELECT column_name1 || column_name2 [,column_name3, ...] FROM table_name

where子句

比较查询

在where子句中可以使用比较运算符对数值、字符值等信息进行查询。比较运算符可分为三类：算数比较运算符、BETWEEN…AND运算符和IN运算符。

算数比较运算符

等于（=）、大于等于（>=）、小于等于（<=）、大于（>）、小于（<）、不等于（!=, <>）、不大于（！>）和不小于(!<)。其语法格式为：

column_name symbol value

BETWEEN… AND

column_name BETWEEN value1 AND value2

IN

column_name IN (value1, value2, value3, ...)

字符串比较

  1. 使用比较运算符（不区分大小写）
  2. 使用BINARY关键字（区分大小写）
    SELECT string1 BINARY symbol string2
    SELECT BINARY string1 symbol string2

日期时间的比较

逻辑查询

keyword：AND, OR, NOT

空值查询

IS NULL OR IS NOT NULL

LIKE操作符

通配符”_”: 匹配单个字符

通配符”%”: 匹配0个或多个字符

转义字符: 通过ESCAPE关键字定义转义字符

SELECT dptName, dptID
FROM T_dpt
WHERE dptID like '$%_%' ESCAPE '$'

使用REGEXP关键字进行模式匹配:

  • ^: 匹配字符串开始。
  • $: 匹配字符串结尾。
  • []: 方括号中的任何字符都可以匹配。
  • -: 表示字符匹配的范围。e.g.: [a-z0-9]。
  • +: 匹配一次或者多次。
  • *: 匹配零次或者多次。
  • (): 括号内的字符看做一个整体。
  • {m}: 匹配m次。

语法格式如下：

string REGEXP pattern_string

表中数据的排序与分组

ORDER BY

  1. 指定列名：

    ORDER BY column_name [ASC|DESC]

    对于指定要排序的列，如果该列中存在空值的行，则在升序排序中，该行会显示在最前面，在降序排序中，该行会显示在最后面。

  2. 指定列的位置序号进行排列：

    ORDER BY column_id [ASC|DESC]

  3. 指定表中的多列进行排序:

    ORDER BY column_name1, column_name2[,column_name3] [ASC|DESC]

常用的聚合函数

1. COUNT(* | column_name | expression)
2. MAX(column_name | expression)
3. MIN(column_name | expression)
4. SUM(column_name | expression)
5. AVG(column_name | expression)

ps:
  COUNT, SUM, AVG可以用DISTINCT关键词去重；
  除了COUNT(*), 其他函数都在计算的时候忽略空值。

GROUP BY

语法规则：

GROUP BY column_name1[, column_name2, ...]
[ORDER BY ...]| [HAVING expression]

ROLLUP

ROLLUP关键词可以对分组后的数据进行阶段性的操作（相当于小计），并对查询的所有数据进行统计（相当于总计）。语法规则如下：

GROUP BY column_name1 WITH ROLLUP

LIMIT

限制查询结果集合的行数。语法规则：

LIMIT n

连接查询和集合查询

内连接查询

定义：将几个表连接起来并查询满足条件的记录。

等值连接

SELECT table_name1.column_name, table_name2.column_name
FROM table_name1, table_name2
WHERE table1.column_name1 = table_name2.column_name2

#alias
SELECT A.column_name, B.column_name
FROM table_name1 A, table_name2 B
WHERE A.column_name1 = B.column_name2

#inner join
SELECT A.column_name, B.column_name
FROM table_name1 A INNER JOIN table_name2 B
ON A.column_name1 = B.column_name2

非等值连接

使用除等号运算符以外的其他运算符将指定条件连接起来而执行的查询操作。

USING子句

SELECT A.column_name, B.column_name
FROM table_name1 A JOIN table_name2
USING (column_name)

外连接查询

外连接查询不仅可以返回满足连接条件的记录，对于一个数据表中的在另一个数据表不匹配的记录也能够返回。

左外连接

返回满足连接条件的记录和左边的所有不满足连接条件的记录。

SELECT table_name1.column_name, table_name2.column_name, ...
FROM table_name1 LEFT JOIN table_name2
ON table_name1.column_name1 = table_name2.column_name2

右外连接

返回满足连接条件的记录和右边的所有不满足连接条件的记录。

SELECT table_name1.column_name, table_name2.column_name, ...
FROM table_name1 RIGHT JOIN table_name2
ON table_name1.column_name1 = table_name2.column_name2

全外连接

返回满足连接条件的记录、左边和右边的所有不满足连接条件的记录。

SELECT table_name1.column_name, table_name2.column_name, ...
FROM table_name1 FULL [OUTER] JOIN table_name2
ON table_name1.column_name1 = table_name2.column_name2

集合查询

并操作

SELECT sentence 1
UNION
SELECT sentence 2

交操作

SELECT sentence 1
INTERSECT
SELECT sentence 2

差操作

SELECT sentence 1
MINUS
SELECT sentence 2

子查询

子查询是指将一个SELECT查询语句嵌套在另一个SQL查询语句中。也称嵌套查询。

单行子查询

子查询的查询语句返回的是一行数据

多行子查询

子查询返回的结果中含有多行数据。

  1. IN 运算符的子查询：

    SELECT ... FROM ... WHERE col_name [NOT] IN (SELECT 子句) [ORDER BY ...]

  2. ANY 运算符的子查询: 只要与子查询的结果集合中的任何一个结果匹配，即可返回。

    SELECT ... FROM ... WHERE col_name op ANY(SELECT 子句) [ORDER BY ...]

  3. ALL 运算符的子查询：必须与子查询的所有结果相匹配，才能返回。

    SELECT ... FROM ... WHERE col_name op ALL(SELECT 子句) [ORDER BY ...]

多列子查询

子查询的语句会返回多个数据列的子查询语句。在多列子查询中，WHERE子句中需要使用括号吧多个属性名称括在一起，用逗号隔开。

SELECT ... FROM ... WHRER (col_name1, col_name2, ...) op (SELECT 子句)

MySQL介绍(1)： 数据库和数据表的创建和管理

发表于 2016-05-07   |   分类于 Dev ， MySQL

数据库的创建和删除

创建数据库

CREATE DATABASE database_name;

删除数据库

DROP DATABASE database_name;

数据表的创建与更新

基本概念

数据表（table）

数据表是存储数据的基本单元，用于描述一个关系。由行和列两部分组成。其中行用来描述实体中的具体数据，一行数据也被称为一条数据记录，或简称记录。字段是表中的一列，用来保存数据的某一特定信息。

主键（primary key）

用于唯一标示一条记录，必须唯一且不能为null。

外键（foreign key）

如果属性列F是关系B的一个属性，同时也是关系A的主键，则称F为关系B的外键。关系A的表称为主表，关系B的表称为主表的从表。

索引（index）

对数据的一个列或者多个列进行索引，加快对数据表的访问。where后面的字段如果有索引，则根据索引进行查找，否则逐条进行扫描。

约束

唯一约束（UNIQUE）：列内的取值不可重复，但可以为空。
主键约束（PRIMARY KEY）：列内的取值唯一且不为空，一个列只能定义一个primary key。
外键约束（FOREIGN KEY）：确保一个表的操作不会对与之关联的表造成不利影响。
检查约束（CHECK）：限制列的取值范围或者取之条件。
非空约束（NOT NULL）：不能为空值。

创建表

CREATE TABLE table_name{
    column_name1 datatype1 [constraint_condition1]
    [, column_name2 datatype2 [constraint_condition2]]
}

使用约束

  • 唯一约束：
    可以定义在表级上，也可以定义在列级上。一般为列创建唯一约束后，数据库会自动为该列建立一个唯一索引。

  • 主键约束：
    使用一个列定义：直接在constraint_condition中添加；
    使用多个列定义：PRIMARY KEY(col_name1[, col_name2])

  • 外键约束：

    FROEIGN KEY[表名\](列名) REFERENCES 表名(列名)
[ON UPDATE [CASCADE\]|[SET NULL\]|[\RESTRICT]\]
[ON DELETE [CASCADE\]|[SET NULL\]|[\RESTRICT]\]

    CASECADE：级联删除。如果主表中的一条数据记录被删除，那么从表中与之相对应的数据也将被一起删除。
    SET NULL：置空删除。如果主表中的一条数据记录被删除，那么从表中与之相对应的数据也将被设置为空值。
    RESTRICT：受限删除。如果主表中的一条数据记录被删除，则在执行DELETE命令时数据库管理系统会报错，提示用户和主表相关联的数据在从表中仍然存在。并不会对从表中的数据进行删除处理。默认方式。

  • 检查约束：
    CHECK(condition)

  • 非空约束：
    [constriant_condition = NOT NULL]

使用索引

索引可以分为唯一索引，主索引，单列索引，复合索引以及聚簇索引等。

  • 唯一索引：在数据表中使用UNIQUE关键字可以为一个数据列定义一个唯一索引。

  • 主索引：在数据表中使用PRIMARY KEY关键字可以为一个数据列定义一个主索引。

  • 单列索引： 定义在一个数据列上的索引。

  • 复合索引：定义在一个数据表的多个数据列上。

  • 聚簇索引： ？

创建和删除索引

创建索引：

CREATE [UINQUE]|[CLUSTER] INDEX index_name ON table_name(column_name [排序方式])

删除索引：

DROP INDEX index_name

修改数据库中的表

增加一列

ALTER TABLE table_name ADD(column_name datatype [constraint_condition])

增加一个约束条件

ALTER TABLE table_name ADD constraint_type(column_name)

增加一个索引

ALTER TABLE table_name ADD INDEX(column_name1 [, column_name2] ...)

修改表中的某一列

ALTER TABLE table_name MODIFY column_name datatype

删除表中的某一列

ALTER TABLE table_name DROP column_name

删除一个约束条件

ALTER TABLE table_name DROP constraint_type

删除数据库中的表

DROP TABLE table_name [CASCADE CONSTRAINTS]

Android四大组件介绍

发表于 2016-05-04   |   分类于 Dev ， Android

简介

Android 系统的四大组件为活动（Activity）、服务（Service）、 广播接收者（Broadcast Receiver）和内容提供者（Content Provider）。
其中活动是所有android应用程序的界面，你所看到的所有android应用里的东西，都是放在activity里面的。Service则是在后台默默运行的进程，即使退出程序，服务也可以仍然运行。广播接收者允许你的应用接受来自其他程序的广播消息，当然你也可以借助它向其他程序广播消息。内容提供者则为应用程序之间的数据共享提供了一种方便且安全的方法。

活动（Activity）

活动的基本用法

活动是一种包含用户界面的组件，主要用于和用户进行交互。一个应用程序中可以包含零个或多个活动。
所有的活动都需要继承Activity类，并重写 Activity 的 onCreate()方法。

public class FirstActivity extends Activity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.id.first_layout);
    }
}

这里调用了 setContentView()方法来给当前的活动加载一个布局，而在 setContentView()方法中，我们一般都会传入一个布局文件的 id。

所有的活动都要在 AndroidManifest.xml 中进行注册才能生效。

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.activitytest"
    android:versionCode="1"
    android:versionName="1.0" >
    <uses-sdk
        android:minSdkVersion="14"
        android:targetSdkVersion="19" />
    <application
        android:allowBackup="true"
        android:icon="@drawable/ic_launcher"
        android:label="@string/app_name"
        android:theme="@style/AppTheme">
        <activity
            android:name=".FirstActivity"
            android:label="This is FirstActivity" >
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>

可以看到，活动的注册声明要放在标签内，这里是通过标签来对活动进行注册的。首先我们要使用android:name 来指定具体注册哪一个活动，然后我们使用了android:label指定活动中标题栏的内容。需要注意的是，给主活动指定的label不仅会成为标题栏中的内容，还会成为启动器(Launcher)中应用程序显示的名称。之后在标签的内部我们加入了标签，并在这个标签里添加了和 这两句声明。如果你想让 FirstActivity 作为我们这个程序的主活动，即点击桌面应用程序图标时首先打开的就是这个活动，那就一定要加入这两句声明。

隐藏标题栏
requestWindowFeature(Window.FEATURE_NO_TITLE);
在活动中使用Toast
Toast.makeText(FirstActivity.this, "You clicked Button 1",
    Toast.LENGTH_SHORT).show();

makeText()方法需要传入三个参数。第一个参数是Context，也就是Toast要求的上下文，由于活动本身就是一个Context对象，因此这里直接传入 FirstActivity.this即可。第二个参数是Toast显示的文本内容，第三个参数是Toast显示的时长，有两个内置常量可以选择Toast.LENGTH_SHORT和 Toast.LENGTH_LONG。

组件和布局的关联
Button button1 = (Button) findViewById(R.id.button_1);

在活动中，可以通过 findViewById()方法获取到在布局文件中定义的元素，findViewById()方法返回的是一个 View 对象，我们需要向下转型将它转成 Button 对象。

在活动中使用Menu

首先在 res 目录下新建一个 menu 文件夹，然后新建一个叫menu的布局文件menu.xml。

<menu xmlns:android="http://schemas.android.com/apk/res/android" >
    <item
        android:id="@+id/add_item"
        android:title="Add"/>
    <item
        android:id="@+id/remove_item"
        android:title="Remove"/>
</menu>

在Activity中重写onCreateOptionsMenu()和onOptionsItemSelected()函数。

@Override
public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.main, menu);
    return true;
}

@Override
public boolean onOptionsItemSelected(MenuItem item) {
    switch (item.getItemId()) {
    case R.id.add_item:
        Toast.makeText(this, "You clicked Add", Toast.LENGTH_SHORT).show();
        break;
    case R.id.remove_item:
        Toast.makeText(this, "You clicked Remove", Toast.LENGTH_SHORT).show();
        break;
    default:
    }
    return true;
}

通过 getMenuInflater()方法能够得到 MenuInflater 对象，再调用它的 inflate()方法就可以给当前活动创建菜单了。inflate()方法接收两个参数，第一个参数用于指定我们通过哪一个资源文件来创建菜单，这里当然传入 R.menu.main，第二个参数用于指定我们的菜单项将添加到哪 一个 Menu 对象当中，这里直接使用 onCreateOptionsMenu()方法中传入的 menu 参数。然后给这个方法返回 true，表示允许创建的菜单显示出来，如果返回了 false，创建的菜单将无法显示。

在 onOptionsItemSelected()方法中，通过调用 item.getItemId()来判断我们点击的是哪一个菜单项，然后给每个菜单项加入自己的逻辑处理。

销毁活动
finish();

只要按一下 Back 键就可以销毁当前的活动了。不过如果你不想通过按键的方式，而是希望在程序中通过代码来销毁活动，当然也可以，Activity 类 供了一 个 finish()方法，我们在活动中调用一下这个方法就可以销毁当前活动了。

Markdown Introduction

发表于 2016-04-25   |   分类于 Dev ， Markdown

区块元素

标题

在MarkDown中，如果一段文字被定义为标题，只需要在文字前面加上#号即可.

#一级标题
##二级标题
###三级标题

以此类推。

引用

Markdown使用类似email方式的>符号来表示引用。在每行面前加上>标记。

> 这是一句引用
> 
> 每句话前面都需要加上>标记

显示的效果如下：

这是一句引用

每句话前面都需要加上>标记

在区块引用可以嵌套，也可以在区块引用里面使用其他的markdown语法。

列表

markdown支持有序列表和无序列表。

无序列表可以使用星号(*)、加号(+)和减号(-)作为列表标记。

+ item1
+ item2
+ item3

显示效果如下：

  • item1
  • item2
  • item3

有序列表则使用数字+英文句点的形式：

1. item1
2. item2
3. item3

显示效果如下：

  1. item1
  2. item2
  3. item3

代码区块

在markdown中建立代码区块很简单，只要简单地使用四个缩进或者一个制表符就可以。

#include<stdio.h>
void main(){
    printf("hello world");
}

代码区块中，一般的markdown语法不会被转换。

分割线

markdown中支持在一行使用三个或以上的星号，减号，底线来建立分割线，行内不能有其他的字符。可以在星号或减号之间插入空格：

***

* * *

---

- - -

_____

显示效果如下：

区段元素

链接

markdown支持行内式和参考式两种形式的链接语法。在两种形式中，链接文字都是采用[方括号]进行标记。

要建立一个行内式的链接，只要在方括号后面紧跟圆括号并插入网址即可。如果还要加上链接的title文字，则直接在网址后面用双引号把title包起来即可。

[百度](http://www.baidu.com)

[百度](http://www.baidu.com "baidu")

显示效果如下：

百度

百度

如果要连接到本地主机的资源，可以使用相对路径。

参考式的链接在方括号后面再跟一个方括号，第二个方括号里面要填上用于辨识链接的标记。

这是一个[参考链接][id1]

然后可以在文件的任意处将该标记的链接内容定义出来。

[id1]: http://www.baidu.com/  "title"

链接辨别标签可以有字母、数字、空白和标点符号，但是并不区分大小写。隐式链接标记功能让你可以省略指定链接标记，这种情形下，链接标记会视为等同于链接文字，要用隐式链接标记只要在链接文字后面加上一个空的方括号。

强调

markdown使用星号(**)和底线()作为标记强调字词的符号。被\或\包围的字词会被转为用标签包围，而如果用两个*或者两个_的话则会被转为用标签包围。

*This is a sentence.*

_This is a sentence._

**This is a sentence.**

__This is a sentence.__

显示效果如下：

This is a sentence.

This is a sentence.

This is a sentence.

This is a sentence.

代码

如果要标记一小段行内的代码，可以使用反引号`把它引起来。

`printf()`可以用来输出。

显示效果如下：

printf()可以用来输出。

图片

markdown使用和链接和相似的方法来标记图片，同样有行内式和参考式两种方法。

行内式：

![alter text](/path/to/image.jpg)
![alter text](/path/to/image.jpg "title")

参考式：

![alter text][id]
[id]: /path/to/image.jpg "title"

自动链接

markdown支持将短的链接或邮箱用<>括号括起来，

<http://www.baidu.com>

http://www.baidu.com

反斜杠

markdown可以利用反斜杠来插入一些在语法中有其它意义的符号，例如：如果你想要用星号加在文字旁边的方式来做出强调效果（但不用 标签），你可以在星号的前面加上反斜杠：

\*literal asterisks\*

+   加号
-   减号

!   惊叹号
